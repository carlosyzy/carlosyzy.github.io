# Boo Engine 官方网站 - 产品需求文档

## 1. 产品概述

Boo Engine 是一款自研的游戏引擎，本网站作为其官方静态展示网站，采用黑科技/赛博朋克风格设计，为用户提供游戏引擎特性展示、文档浏览、版本历史和示例体验等功能。

**核心目标：**
- 展示游戏引擎的强大功能和技术优势
- 提供清晰的文档导航和版本信息
- 通过示例让用户快速了解引擎能力

## 2. 功能模块

### 2.1 页面结构

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| **Home（首页）** | Hero区域 | 引擎特性展示，交替左右布局 |
| **Docs（文档）** | 导航侧边栏+内容区 | 现有文档系统迁移，保持功能 |
| **History（历史）** | 版本列表 | 显示所有发布版本，最新在上 |
| **Examples（示例）** | 示例网格+弹窗 | 网格展示示例，点击弹窗播放 |

### 2.2 页面详情

#### 2.2.1 Home 首页

**Hero区域 - 引擎特性展示：**
- **布局策略**：左右交替展示，奇数项（1,3,5...）左图右文，偶数项（2,4,6...）左文右图
- **内容项**：
  1. C++ 高性能核心 - 图标+文字描述
  2. Vulkan 渲染 API - 文字+图片
  3. 跨平台支持 - 图标+文字描述（Windows, macOS, Android）
  4. 2D/3D 渲染 - 文字+图片
  5. 实时物理引擎 - 图标+文字描述
  6. 资源管理系统 - 文字+图片

**设计要点：**
- 每个特性项使用统一的卡片样式
- 包含图标、标题、详细描述
- 支持动画效果（入场动画、悬停效果）

#### 2.2.2 Docs 文档

**功能：**
- 继承现有文档系统功能
- 左侧导航菜单，右侧内容区域
- 支持搜索、层级展开
- 通过 iframe 加载文档内容

**导航结构：**
```
- 入门指南
- 场景系统
- 组件系统
- 资源系统
- 声音系统
- 跨平台
```

#### 2.2.3 History 历史

**功能：**
- 显示所有已发布版本信息
- 最新版本永远在最上方
- 支持版本配置（版本号、发布日期、特性列表、下载地址）

**配置结构：**
```javascript
{
  version: "1.0.0",
  date: "2024-01-15",
  features: ["特性1", "特性2"],
  download: "https://..."
}
```

**显示效果：**
- 版本卡片形式展示
- 包含版本号、发布日期、关键特性
- 提供下载链接

#### 2.2.4 Examples 示例

**功能：**
- 网格布局展示示例（一行3-4个）
- 默认显示预览图、名称、描述
- 点击示例卡片弹出弹窗
- 弹窗显示示例配置和播放按钮

**弹窗内容：**
- 示例预览/配置信息
- 开始播放按钮
- 关闭按钮

**数据结构：**
```javascript
{
  name: "示例名称",
  description: "示例描述",
  thumbnail: "预览图路径",
  config: { /* 示例配置 */ },
  download: "下载链接"
}
```

## 3. 设计规范

### 3.1 整体风格

**主题：黑科技 / 赛博朋克 / 黑客帝国风格**

| 设计元素 | 规范 |
|---------|------|
| **背景色** | 深黑色 (#0a0a0f, #111111) |
| **主色调** | 科技红 (#ff0040, #ff3366) |
| **辅助色** | 科技蓝 (#00d4ff, #0099ff)、科技绿 (#00ff88, #00cc66) |
| **文字色** | 主文字白色 (#ffffff)，次要文字灰色 (#888888) |
| **边框/分割** | 使用渐变色边框，模拟霓虹效果 |

### 3.2 布局规范

**容器宽度：**
- 最大宽度：1400px
- 左右内边距：40px
- 上下内边距：60px

**栅格系统：**
- 12列栅格
- 响应式：桌面4列 → 平板3列 → 手机2列

### 3.3 视觉元素

**图标：**
- 使用简约几何图标
- 带发光/霓虹效果
- 颜色：红、蓝、绿主色调

**字体：**
- 主字体：无衬线字体，带科技感
- 代码字体：等宽字体

**动效：**
- 入场动画：渐入+上滑，stagger效果
- 悬停效果：发光增强、缩放、颜色变化
- 过渡动画：0.3s ease-out

**装饰元素：**
- 网格线背景
- 扫描线效果
- 渐变光晕
- 几何图形装饰

## 4. 技术实现

### 4.1 技术栈

- **HTML5 + CSS3 + Vanilla JavaScript**
- **无需框架**：纯静态网站，快速加载
- **外部依赖**：
  - Marked.js（文档渲染）
  - Google Fonts（字体）

### 4.2 文件结构

```
Boo-Engine-H5/
├── index.html              # 主入口
├── css/
│   └── main.css           # 全局样式
├── js/
│   ├── app.js             # 主应用逻辑
│   ├── home.js            # 首页模块
│   ├── docs.js            # 文档模块
│   ├── history.js         # 历史版本模块
│   └── examples.js        # 示例模块
├── pages/
│   ├── home.html          # 首页
│   ├── docs.html          # 文档页
│   ├── history.html       # 历史页
│   └── examples.html      # 示例页
├── images/
│   ├── icons/            # 特性图标
│   └── examples/         # 示例预览图
└── docs/                  # 文档内容（已存在）
```

### 4.3 路由设计

| 路由 | 页面 | 功能 |
|------|------|------|
| /index.html | 首页 | 引擎特性展示 |
| /pages/docs.html | 文档 | API文档浏览 |
| /pages/history.html | 历史 | 版本信息展示 |
| /pages/examples.html | 示例 | 示例展示与播放 |

## 5. 用户流程

```
用户访问网站
    ↓
首页（引擎介绍）
    ↓
顶部导航栏 → 切换模块
    ├── Docs → 文档页面
    ├── History → 版本历史
    └── Examples → 示例展示
```

## 6. 响应式设计

**断点：**
- 桌面：≥1024px（4列网格）
- 平板：768px-1023px（3列网格）
- 手机：<768px（2列网格，单列布局）

**适配策略：**
- 移动端顶部导航变为汉堡菜单
- 内容区域全宽显示
- 保持可读性和可操作性
